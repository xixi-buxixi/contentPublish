# Pulse Distro 项目使用说明书

本说明书将引导您从零开始，在本地环境运行并体验 Pulse Distro 的完整发布分发流程。

---

## 🛠️ 准备工作

1. **运行环境**：确保本地已安装 JDK 21 和 Maven 3.x。
2. **配置文件**：在根目录下创建 `.env` 文件并装填您的配置：
   ```properties
   SERVER_PORT=8080
   APP_PUBLIC_BASE_URL=http://localhost:8080
   # 如果需要使用 DeepSeek 进行智能重写，请填入真实的 API Key，否则系统会自动降级为模板适配
   LANGCHAIN4J_API_KEY=sk-xxxxxx
   LANGCHAIN4J_BASE_URL=https://api.deepseek.com
   LANGCHAIN4J_MODEL_NAME=deepseek-chat
   ```
3. **启动应用**：
   ```bash
   mvn spring-boot:run
   ```
   应用启动后，使用浏览器打开控制台首页 `http://localhost:8080`。

---

## 🏃 核心操作流程

```
[第一步：会话初始化] ──► [第二步：新建任务] ──► [第三步：补充配图]
                                                        │
┌───────────────────────────────────────────────────────┘
▼
[第四步：智能适配 (AI/模板)] ──► [第五步：拟态样板间预览] ──► [第六步：仿真发布]
```

### 第一步：会话初始化与连接
1. 打开浏览器进入系统首页。
2. 页面加载时，前端将自动调用后端接口 `POST /api/session/init`。
   * **后端行为**：如果没有会话，会生成一个 UUID 形式的 `userToken` 返回给前端，并建立 WebSocket 连接。
   * **界面反馈**：控制台右上角显示“WebSocket 已连接”，这表示实时事件通知管道已就绪。

### 第二步：新建任务与正文标准化
1. 在控制台的“新建任务”卡片中输入文章标题（如：《高并发系统设计实践》）。
2. 在正文输入框中，写入或粘贴含有 Markdown 格式的原始正文。
3. 点击 **“解析标准化”** 按钮。
   * **后端行为**：系统利用 Flexmark 解析 Markdown 语法，自动将第一段提取为“内容摘要”，并将正文拆分为结构化的“内容块 JSON”（标题块、段落块、列表块、图片块）。此时任务状态为 `PENDING`。
   * **界面反馈**：页面左下侧生成标准的文档积木块列表。

### 第三步：上传任务配图与素材池匹配
1. 在正文 Markdown 中如果含有本地图片占位符（如 `![架构图](media-placeholder)`）。
2. 在底部的“素材池”区域，点击 **“上传图片”**，上传本地 PNG 或 JPEG 图片。
   * **后端行为**：调用 `POST /api/tasks/{taskId}/media`。系统计算图片哈希值并存储到 `data/media/{taskId}/` 下。
3. 如果需要修改正文图片，在标准块编辑器中将图片块的 `mediaId` 修改为上传后生成的图片 ID，并点击“更新”。
   * **后端行为**：调用 `PUT /api/tasks/{taskId}/normalized` 强校验图片归属权并补齐宽高等元数据。
   * **界面反馈**：草稿内图片成功加载显示。

### 第四步：多平台智能适配 (异步)
1. 在适配平台面板中，勾选需要同步分发的平台（支持：`小红书`、`知乎`、`微信公众号`、`B站`）。
2. 点击 **“开始智能适配”**。
   * **后端行为**：调用 `POST /api/tasks/{taskId}/adapt`。任务状态转为 `ADAPTING`，后端立刻向前端返回 `202 Accepted`。
   * **异步改写**：各平台适配任务提交到后台线程池执行。如果配有 DeepSeek 密钥，大模型会根据各平台的字数、风格要求进行改写；如无 Key 或网络超时，则自动降级为本地模板强行截断，并设定预设标签，同时发送降级事件。
   * **界面反馈**：右侧各平台 Tab 页处于“转圈 Loading”状态。一旦某一平台适配完成，通过 WebSocket 收到通知，Tab 变绿就绪。若使用了降级模板，Tab 上会显示黄色感叹号标徽。

### 第五步：拟态“样板间”效果预览与人工修改
1. 点击已就绪的平台 Tab 页（例如“小红书”）。
2. 点击 **“效果预览”** 按钮。
   * **后端行为**：页面打开 `GET /mock/xiaohongshu/{recordId}`。后端拼装带有 Tailwind v4 的仿真移动端 HTML5 页面。
   * **界面反馈**：展示出带有仿真手机壳、状态栏、点赞数据的小红书卡片预览。
3. 如果发现内容格式有微瑕，在 Tab 页的文本框中直接进行修改，然后点击 **“保存修改”**。
   * **后端行为**：调用 `PUT /api/records/{recordId}` 保存人工修正后的内容。

### 第六步：一键仿真发布
1. 确认无误后，点击工作台右下角的 **“仿真发布 (Mock Publish)”**。
   * **后端行为**：调用 `POST /api/tasks/{taskId}/publish`，状态置为 `SUCCESS`，记录发布时间，生成最终访问的发布 Mock 地址。
   * **界面反馈**：控制台弹出“发布成功”通知，状态卡片变为绿色，提供各平台最终的发布成果仿真链接。
