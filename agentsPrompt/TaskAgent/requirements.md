# Content Task Module Requirements (内容任务模块需求)

## 1. 内容任务数据库设计 (content_task)
需创建 `content_task` 表用于持久化原始内容与标准化解析内容：
```sql
CREATE TABLE content_task (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    source_type VARCHAR(50) NOT NULL,            -- TEXT, MARKDOWN 等
    raw_content CLOB NOT NULL,                    -- 原始内容
    normalized_content_json CLOB,                 -- 标准化内容模型 JSON
    cover_media_id BIGINT,                        -- 封面图媒体 ID
    status VARCHAR(50) NOT NULL,                  -- PENDING, READY, FAILED 等
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 2. 标准内容数据结构 (Normalized Content Model)
在内存中定义以下结构（推荐使用 Java Record）：
*   **MediaRef**：媒体引用对象。
    *   `mediaId` (Long): 媒体资源主键。
    *   `publicUrl` (String): 可被浏览器直接读取的图片 URL。
    *   `alt` (String): 替换文本。
    *   `width` (Integer), `height` (Integer)
*   **ContentBlock**：内容块。
    *   `type` (String): heading, paragraph, list, image。
    *   `level` (Integer): 标题层级（适用于 heading）。
    *   `text` (String): 文本内容（适用于段落/标题/列表）。
    *   `media` (MediaRef): 媒体引用（适用于 image）。
*   **NormalizedContent**：整合标准内容。
    *   `title` (String): 统一标题。
    *   `summary` (String): 摘要提取。
    *   `blocks` (List<ContentBlock>): 有序的内容块列表。
*   **AdaptedContent**：平台适配后的正文。
    *   `platform` (String), `title` (String), `content` (String), `tags` (List<String>), `media` (List<MediaRef>), `styleExplanation` (String)。

## 3. 解析与更新规则
*   **Markdown 解析**：能够识别标题（`#`、`##` 等）、普通段落、无序/有序列表以及图片节点 `![alt](url)`。
*   **图片处理**：图片在解析时只保存 `mediaId` 和 `publicUrl`。禁止将前端产生的本地临时路径直接写入最终的标准内容 JSON 中。
*   **一致性约束**：前端更新标准内容模型时，后端应确保正文中的 `mediaId` 归属于当前任务，并且状态为 `READY`，在保存前重新查询数据库补充 URL 冗余。
