# Platform Config Module Requirements (平台配置模块需求)

## 1. 平台配置表设计 (platform_config)
```sql
CREATE TABLE platform_config (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    platform VARCHAR(50) NOT NULL UNIQUE,          -- 平台标识，如 xiaohongshu
    display_name VARCHAR(100) NOT NULL,           -- 显示名，如 小红书
    config_json CLOB NOT NULL,                    -- 配置细则 JSON
    enabled BOOLEAN NOT NULL DEFAULT TRUE,        -- 开关状态
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 2. 配置文件格式 (platforms/*.json)
每个平台的具体发布规则以独立 JSON 文件形式放在 classpath 的 `platforms/` 目录下（如 `xiaohongshu.json`）：
```json
{
  "platform": "xiaohongshu",
  "displayName": "小红书",
  "maxTitleLength": 20,
  "maxContentLength": 1000,
  "maxTags": 10,
  "supportsMarkdown": false,
  "image": {
    "maxCount": 9,
    "acceptedMimeTypes": ["image/jpeg", "image/png", "image/webp"]
  },
  "stylePrompt": "改写为口语化、轻量、适合种草分享的风格，保留核心观点，增加自然 Emoji 和话题标签。"
}
```

## 3. 初始化与同步逻辑
*   **启动数据刷入**：应用在初始化（如实现 `CommandLineRunner`）时，需要自动读取 `resources/platforms/` 下所有的平台配置 JSON，并将其同步更新到数据库表 `platform_config` 中，支持覆盖更新。
*   **动态控制**：前端在渲染渠道时，只拉取 `enabled = true` 且处于开启状态的平台。
