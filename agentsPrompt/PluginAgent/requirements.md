# Chrome Extension Module Requirements (浏览器插件模块需求)

## 1. 插件会话表设计 (plugin_session)
```sql
CREATE TABLE plugin_session (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id BIGINT NOT NULL,                      -- 绑定用户 ID
    session_id VARCHAR(100) NOT NULL,             -- 插件生成的会话 ID
    extension_version VARCHAR(50),                -- 插件版本
    browser VARCHAR(100),                         -- 浏览器名称 (Chrome 等)
    status VARCHAR(50) NOT NULL,                  -- ONLINE, OFFLINE
    last_heartbeat_at TIMESTAMP,                  -- 上次心跳时间
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 2. 插件注册与保活机制
*   **注册匹配**：插件注册时必须携带 Header 中的 `X-User-Token`。后端利用 `userToken` 加 `sessionId` 作为唯一标识进行绑定注册。若同一 `userToken` 注册了新的 `sessionId`，可将原 `sessionId` 强制踢下线并标志为 `OFFLINE`。
*   **高频心跳**：插件每隔 15~30 秒发起一次保活心跳包。
*   **心跳超时检测**：若最近一次心跳时间与当前时间差值大于 60 秒，系统应视其为超时下线（`OFFLINE`）。在工作台发起真实发布前，必须检测该状态。
*   **发布状态接管与挂起**：插件在辅助执行填表发布时，如果遇到验证码或账号异常，需上报 `status=SUSPENDED` 并提供验证码界面截图或异常日志，以便让工作台提示创作者进行人工介入。
